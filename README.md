# VioletScreen

Веб-платформа для просмотра фильмов, сериалов, аниме и мультфильмов.

## Технологии
- HTML5
- CSS3
- JavaScript (ES6)
- localStorage для хранения данных
- JSON для статического хостинга

## Структура проекта
```
violet-screen/
├── index.html
├── admin.html
├── style.css
├── script.js
├── data.json
├── assets/
│   ├── posters/
│   └── videos/
└── README.md
```

## Запуск
1. Откройте `index.html` в браузере
2. Для добавления контента перейдите на `admin.html`

## Функционал
- Просмотр фильмов и сериалов
- Поиск по названию и жанрам
- Добавление контента через админ-панель
- Хранение данных в localStorage
- Адаптивный дизайн
- Поддержка JSON-файла для статического хостинга

## Работа с данными

### Локальное хранение
Данные сохраняются в localStorage браузера. Все изменения, внесенные через админ-панель, сохраняются локально на устройстве пользователя.

### Статический хостинг с JSON
Для развертывания на статических хостингах (GitHub Pages, Netlify, Vercel):

1. **Загрузка данных**:
   - Сайт сначала пытается загрузить данные из `data.json`
   - При ошибке загрузки используется резервный вариант из localStorage
   - Если данных нет ни в JSON, ни в localStorage, используются демо-данные

2. **Обновление данных**:
   - При редактировании контента через админ-панель изменения сохраняются только в localStorage
   - Для публикации изменений на сервере необходимо вручную обновить файл `data.json`
   - Рекомендуется экспортировать данные из localStorage и заменить ими содержимое `data.json`

3. **Экспорт данных**:
   - Откройте консоль браузера (F12)
   - Выполните команду: `localStorage.getItem('violetScreenContent')`
   - Скопируйте результат и вставьте в файл `data.json`

## Развертывание

### На статическом хостинге
1. Подготовьте файл `data.json` с актуальными данными
2. Загрузите все файлы проекта на хостинг
3. При необходимости обновления контента:
   - Внесите изменения через админ-панель
   - Экспортируйте данные из localStorage
   - Обновите файл `data.json` на сервере

### Альтернативные решения
- Настройка CI/CD для автоматического обновления `data.json`
- Использование GitHub Actions для синхронизации данных
- Интеграция с Google Sheets через API
